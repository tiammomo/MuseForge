# 商品事实真实性规则

## 目标

将原始图片、商品介绍、参数和说明文档转换为可验证的商品事实。不要用常识补齐缺失参数，不要把营销表达当成结构事实。

## 必须锁定

- 商品名称、SKU、数量和包装内容。
- 商品形状、颜色、材料、表面、结构和可见部件。
- 有明确依据的单件尺寸、整体尺寸、重量和排列方式。
- 有明确依据的安装、使用、维护和保护方式。
- 主商品与配件真实存在的相对大小关系。

## 冲突处理

1. 原始商品资料中的明确参数优先于图片推测。
2. 多份文档冲突时，在 prompt 中省略冲突字段，不自行选择。
3. 尺寸图只使用明确出现的尺寸；未知尺寸不得生成数字。
4. 参考图无法证明的材质、功能、性能、接口、认证或适用场景不得添加。
5. 不得把包装外尺寸误写成商品本体尺寸。

## 输出要求

在每条 prompt 中保留简洁的 `product_information`，只写与当前图片类型有关的事实。商品事实始终优先于场景创意和视觉装饰。
